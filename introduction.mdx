FlashVote lets you create polls and watch votes roll in live — no page refresh needed. It combines a REST API for poll management with a SignalR hub that pushes vote updates to every connected client the moment a vote is cast.

## What FlashVote does

FlashVote is a two-app monorepo:

- **FlashVoteBackend** — ASP.NET Core Web API with SignalR, Entity Framework Core, and SQLite
- **FlashVoteFrontend** — React 19 + TypeScript app with TanStack Query, SignalR client, Recharts, and Tailwind CSS

When a user votes, the backend persists the vote to SQLite, then broadcasts a `ReceiveVote` event over SignalR to every client watching the same poll. The frontend receives the event, invalidates its React Query cache, and re-fetches the updated results — so live bar charts update for all viewers simultaneously.

## Tech stack

| Layer | Technology |
|---|---|
| Backend framework | ASP.NET Core (.NET 10) |
| ORM / database | Entity Framework Core + SQLite |
| Real-time | SignalR |
| Frontend framework | React 19 + TypeScript + Vite |
| Data fetching | TanStack React Query |
| Real-time client | @microsoft/signalr |
| Charts | Recharts |
| Styling | Tailwind CSS + shadcn-style components |
