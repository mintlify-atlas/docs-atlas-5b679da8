FlashVote uses SignalR to push vote updates to all connected clients watching the same poll. When a vote is cast, the backend broadcasts a `ReceiveVote` event to every client in the poll's SignalR group. Clients subscribe by calling `JoinPoll` after connecting.

## Hub endpoint

```
/pollhub
```

Default local URL: `http://localhost:5000/pollhub`

## Connection approach

The frontend wraps the SignalR `HubConnection` in a `PollRealtimeClient` singleton exported as `pollRealtimeClient`. A single connection is shared across the entire app session — calling `connect()` more than once reuses the existing connection if it is not in the `Disconnected` state.

The connection is built with `withAutomaticReconnect()`, which handles transient network failures without requiring application code to manually reconnect.

<Note>
  `pollRealtimeClient` is a module-level singleton. All components and hooks share the same underlying `HubConnection`. Listeners are added and removed via `addVoteListener` and `addStateListener`, which each return an unsubscribe function.
</Note>

### Client connection code

```typescript
import {
  HubConnection,
  HubConnectionBuilder,
  HubConnectionState,
  LogLevel,
} from '@microsoft/signalr'
import { getApiBaseUrl } from '@/services/http'

type VoteListener = (optionId: string) => void
type StateListener = (state: HubConnectionState) => void

class PollRealtimeClient {
  private connection: HubConnection | null = null
  private voteListeners = new Set<VoteListener>()
  private stateListeners = new Set<StateListener>()

  getConnectionState() {
    return this.connection?.state ?? HubConnectionState.Disconnected
  }

  addVoteListener(listener: VoteListener) {
    this.voteListeners.add(listener)
    return () => this.voteListeners.delete(listener)
  }

  addStateListener(listener: StateListener) {
    this.stateListeners.add(listener)
    return () => this.stateListeners.delete(listener)
  }

  async connect() {
    if (!this.connection) {
      this.connection = new HubConnectionBuilder()
        .withUrl(`${getApiBaseUrl()}/pollhub`, {
          withCredentials: true,
        })
        .withAutomaticReconnect()
        .configureLogging(LogLevel.Warning)
        .build()

      this.connection.on('ReceiveVote', (optionId: string) => {
        this.voteListeners.forEach((listener) => listener(optionId))
      })

      this.connection.onclose(() => this.emitState())
      this.connection.onreconnecting(() => this.emitState())
      this.connection.onreconnected(() => this.emitState())
    }

    if (this.connection.state === HubConnectionState.Disconnected) {
      await this.connection.start()
      this.emitState()
    }

    return this.connection
  }

  async joinPoll(pollId: string) {
    const connection = await this.connect()
    await connection.invoke('JoinPoll', pollId)
  }

  async leavePoll(pollId: string) {
    if (!this.connection || this.connection.state !== HubConnectionState.Connected) {
      return
    }
    await this.connection.invoke('LeavePoll', pollId)
  }

  private emitState() {
    const state = this.getConnectionState()
    this.stateListeners.forEach((listener) => listener(state))
  }
}

export const pollRealtimeClient = new PollRealtimeClient()
```

## Connection lifecycle

<Steps>
  <Step title="Connect to the hub">
    Call `pollRealtimeClient.connect()`. If no `HubConnection` exists yet, one is created pointing at `/pollhub` with `withAutomaticReconnect` enabled. If a connection already exists and is not `Disconnected`, this is a no-op.

    ```typescript
    await pollRealtimeClient.connect()
    ```
  </Step>

  <Step title="Join a poll group">
    Call `joinPoll(pollId)` with the poll's `id` (a UUID string). This invokes the `JoinPoll` hub method on the server, which adds the connection to the SignalR group named after that UUID.

    ```typescript
    await pollRealtimeClient.joinPoll(poll.id)
    ```
  </Step>

  <Step title="Handle ReceiveVote events">
    Register a vote listener before or after joining. The listener receives the `optionId` of the option that was just voted on.

    ```typescript
    const unsubscribe = pollRealtimeClient.addVoteListener((optionId) => {
      console.log('Vote cast for option:', optionId)
    })
    ```
  </Step>

  <Step title="Leave the poll group">
    Call `leavePoll(pollId)` when the user navigates away from the poll. The server removes the connection from the group so it no longer receives `ReceiveVote` broadcasts for that poll.

    ```typescript
    await pollRealtimeClient.leavePoll(poll.id)
    ```
  </Step>

  <Step title="Unsubscribe listeners">
    Call the unsubscribe function returned by `addVoteListener` (and `addStateListener`) to clean up when the component unmounts.

    ```typescript
    unsubscribe()
    ```
  </Step>
</Steps>

## Client-to-server methods

### JoinPoll

Adds the calling connection to the SignalR group for the specified poll. After joining, the connection receives all `ReceiveVote` broadcasts for that poll.

| Parameter | Type   | Required | Description |
|-----------|--------|----------|-------------|
| `pollId`  | string | Yes      | The `id` of the poll to join, formatted as a UUID string (e.g. `"d3b6a1e2-4f5c-4a2b-9e7d-1c2b3d4e5f6a"`). |

```typescript
await pollRealtimeClient.joinPoll(pollId)
// internally: connection.invoke('JoinPoll', pollId)
```

### LeavePoll

Removes the calling connection from the SignalR group for the specified poll. The connection will no longer receive `ReceiveVote` events for that poll.

| Parameter | Type   | Required | Description |
|-----------|--------|----------|-------------|
| `pollId`  | string | Yes      | The `id` of the poll to leave, formatted as a UUID string. |

```typescript
await pollRealtimeClient.leavePoll(pollId)
// internally: connection.invoke('LeavePoll', pollId)
```

<Warning>
  `leavePoll` is a no-op if the connection is not in the `Connected` state. Always call it before disposing a component that called `joinPoll`, but handle the case where the connection may have dropped.
</Warning>

## Server-to-client events

### ReceiveVote

Broadcast by the server to every client in a poll's group immediately after a vote is successfully persisted. The payload is the `id` of the option that received the vote.

| Parameter  | Type   | Description                                         |
| ---------- | ------ | --------------------------------------------------- |
| `optionId` | string | UUID of the `Option` that was voted on.             |

The server sends this event from `PollService.VoteAsync`:

```csharp
await _hubContext.Clients.Group(pollId.ToString())
    .SendAsync("ReceiveVote", optionId);
```

## Group-based architecture

Each poll has its own dedicated SignalR group. The group name is the poll's `Id` serialized as a string (e.g. `"d3b6a1e2-4f5c-4a2b-9e7d-1c2b3d4e5f6a"`). A client that is viewing multiple polls simultaneously would join multiple groups.

When a vote is cast via `POST /api/poll/{pollId}/vote/{optionId}`, the `PollService` persists the increment and then broadcasts `ReceiveVote` only to the group identified by `pollId`. Clients watching other polls are unaffected.

## Registering and unregistering the ReceiveVote handler

The `usePollRealtime` hook shows the recommended React pattern for managing the listener lifecycle:

```typescript
import { HubConnectionState } from '@microsoft/signalr'
import { useQueryClient } from '@tanstack/react-query'
import { useEffect, useRef, useState } from 'react'
import { pollKeys } from '@/lib/query-keys'
import { pollRealtimeClient } from '@/services/signalr'

export function usePollRealtime(activePollId: string | null) {
  const queryClient = useQueryClient()
  const previousPollIdRef = useRef<string | null>(null)
  const activePollIdRef = useRef<string | null>(activePollId)
  const [connectionState, setConnectionState] = useState<HubConnectionState>(
    pollRealtimeClient.getConnectionState(),
  )

  useEffect(() => {
    activePollIdRef.current = activePollId
  }, [activePollId])

  useEffect(() => {
    const unsubscribeState = pollRealtimeClient.addStateListener((state) => {
      setConnectionState(state)
    })

    const unsubscribeVote = pollRealtimeClient.addVoteListener(() => {
      const currentPollId = activePollIdRef.current
      if (!currentPollId) {
        return
      }

      void queryClient.invalidateQueries({ queryKey: pollKeys.detail(currentPollId) })
      void queryClient.invalidateQueries({ queryKey: pollKeys.all })
    })

    return () => {
      unsubscribeState()
      unsubscribeVote()
    }
  }, [queryClient])

  useEffect(() => {
    const syncGroups = async () => {
      if (!activePollId) {
        if (previousPollIdRef.current) {
          await pollRealtimeClient.leavePoll(previousPollIdRef.current)
          previousPollIdRef.current = null
        }
        return
      }

      if (previousPollIdRef.current && previousPollIdRef.current !== activePollId) {
        await pollRealtimeClient.leavePoll(previousPollIdRef.current)
      }

      await pollRealtimeClient.joinPoll(activePollId)
      previousPollIdRef.current = activePollId
    }

    void syncGroups()
  }, [activePollId])

  return {
    connectionState,
  }
}
```

Key points from this hook:

- Vote and state listeners are registered once on mount and cleaned up on unmount via the returned unsubscribe functions.
- When `activePollId` changes, the hook automatically leaves the previous poll group and joins the new one.
- On `ReceiveVote`, the hook invalidates the React Query cache for the active poll, triggering a refetch of updated vote counts — it does not use the `optionId` payload directly.

## Complete integration example

The following example shows a minimal component that connects to the hub, joins a poll group, and reacts to incoming vote events.

```typescript
import { useEffect, useState } from 'react'
import { pollRealtimeClient } from '@/services/signalr'

function PollWatcher({ pollId }: { pollId: string }) {
  const [lastVotedOption, setLastVotedOption] = useState<string | null>(null)

  useEffect(() => {
    // Join the poll's SignalR group
    void pollRealtimeClient.joinPoll(pollId)

    // Register a listener for incoming votes
    const unsubscribe = pollRealtimeClient.addVoteListener((optionId) => {
      setLastVotedOption(optionId)
    })

    return () => {
      // Clean up: leave the group and remove the listener
      void pollRealtimeClient.leavePoll(pollId)
      unsubscribe()
    }
  }, [pollId])

  return (
    <div>
      {lastVotedOption
        ? `Last vote cast for option: ${lastVotedOption}`
        : 'Waiting for votes...'}
    </div>
  )
}
```
