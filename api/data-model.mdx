title: Data model
description: Reference for FlashVote's core entities, DTOs, and their relationships as persisted by EF Core.

FlashVote's data layer has two entities — `Poll` and `Option` — and two DTOs used for creating and updating polls. All IDs are GUIDs. The database is SQLite, managed through Entity Framework Core.

## Relationships

A `Poll` owns a collection of `Option` records. Each `Option` carries a `PollId` foreign key that references its parent `Poll`. The EF Core model configures this as a one-to-many relationship with cascade delete: removing a `Poll` automatically removes all of its `Option` rows.

In prose: one poll → many options; each option belongs to exactly one poll.

    ### Option

    Represents a single voting choice within a poll.

    <ResponseField name="Id" type="Guid" required>

## EF Core DbContext

`ApplicationDbContext` registers both entities and configures the one-to-many relationship between `Poll` and `Option`:

```csharp
public class ApplicationDbContext : DbContext
{
    public ApplicationDbContext(DbContextOptions<ApplicationDbContext> options) : base(options)
    {
    }

    public DbSet<Poll> Polls => Set<Poll>();
    public DbSet<Option> Options => Set<Option>();

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        base.OnModelCreating(modelBuilder);
        modelBuilder.Entity<Poll>()
            .HasMany(p => p.Options)
            .WithOne()
            .HasForeignKey(o => o.PollId)
            .OnDelete(DeleteBehavior.Cascade);
    }
}
```

<Info>
  The `WithOne()` call has no argument because `Option` does not declare an explicit navigation property back to `Poll`. EF Core resolves the relationship through the `PollId` foreign key alone.
</Info>

## Frontend TypeScript types

The frontend mirrors the API shapes with these interfaces:

```typescript
export interface PollOption {
  id: string
  text: string
  voteCount: number
  pollId: string
}

export interface PollListItem {
  id: string
  title: string
  description: string
  createdAt: string
  expiresAt: string | null
  isClosed: boolean
}

export interface PollDetail extends PollListItem {
  options: PollOption[]
}

export interface VoteParams {
  pollId: string
  optionId: string
}
```

`PollDetail` extends `PollListItem` to add the `options` array, matching the difference between the list and detail API responses.
