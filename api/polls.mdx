---
title: Polls
description: REST endpoints for creating and retrieving FlashVote polls.
---

These endpoints let you list all polls, fetch a single poll with its options and vote counts, and create a new poll.

---

## GET /api/poll

Returns all polls stored in the database. Each poll includes its options and current vote counts.

### Response

`200 OK` — array of poll objects.

<ResponseField name="[]" type="object[]">
  <Expandable title="Poll object properties" defaultOpen>
    <ResponseField name="id" type="string" required>
      Unique identifier for the poll (UUID v4).
    </ResponseField>
    <ResponseField name="title" type="string" required>
      Title of the poll.
    </ResponseField>
    <ResponseField name="description" type="string" required>
      Optional descriptive text for the poll.
    </ResponseField>
    <ResponseField name="createdAt" type="string" required>
      ISO 8601 UTC timestamp of when the poll was created.
    </ResponseField>
    <ResponseField name="expiresAt" type="string | null" required>
      ISO 8601 UTC expiry timestamp, or `null` if the poll has no expiry.
    </ResponseField>
    <ResponseField name="isClosed" type="boolean" required>
      Whether the poll has been manually closed.
    </ResponseField>
    <ResponseField name="options" type="object[]" required>
      <Expandable title="Option object properties">
        <ResponseField name="id" type="string" required>
          Unique identifier for the option (UUID v4).
        </ResponseField>
        <ResponseField name="text" type="string" required>
          Display text for the option.
        </ResponseField>
        <ResponseField name="voteCount" type="number" required>
          Number of votes this option has received.
        </ResponseField>
        <ResponseField name="pollId" type="string" required>
          ID of the poll this option belongs to.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

### Examples

          },
          {
            "id": "b2c3d4e5-2222-3333-4444-bbccddeeff00",

## GET /api/poll/\{id\}

Returns a single poll by its UUID, including all options and their current vote counts.

### Path parameters

<ParamField path="id" type="string" required>
  The UUID of the poll to retrieve.
</ParamField>

### Response

`200 OK` — a single poll object (same schema as the items in the list above).

`404 Not Found` — no poll with the given ID exists.

## POST /api/poll

Creates a new poll. The backend generates UUIDs for the poll and each option, sets `CreatedAt` to the current UTC time, and sets `IsClosed` to `false`.

### Request body

<ParamField body="title" type="string" required>
  Title of the poll.
</ParamField>

<ParamField body="description" type="string">
  Optional description for the poll. Defaults to an empty string if omitted.
</ParamField>

<ParamField body="expiresAt" type="string | null">
  ISO 8601 UTC expiry timestamp. Pass `null` or omit to create a poll with no expiry.
</ParamField>

<ParamField body="options" type="string[]" required>
  Array of option text strings. Each string becomes a voting option with `VoteCount` starting at zero.
</ParamField>

### Response

`201 Created` — the newly created poll object, identical in shape to the response from `GET /api/poll/{id}`. The `Location` header points to the new resource URL.

`400 Bad Request` — poll creation failed. Body contains `{ "message": "..." }`.

### Examples
