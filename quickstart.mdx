---
title: Quickstart
description: Get FlashVote running locally in minutes — backend API, SignalR hub, and React frontend.
---

This guide walks you through running the full FlashVote stack on your local machine and verifying that real-time vote updates work end-to-end.

<Steps>
  <Step title="Prerequisites">
    Make sure you have the following installed before you begin:

    | Tool | Required version |
    |---|---|
    | .NET SDK | 10 |
    | Node.js | 20+ |
    | npm | 10+ |

    Verify your versions:

    ```bash
    dotnet --version
    node --version
    npm --version
    ```
  </Step>

  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/R0N7w7/FlashVote.git
    cd FlashVote
    ```
  </Step>

  <Step title="Start the backend">
    From the `FlashVoteBackend/` directory, restore dependencies, build, and run:

    ```bash
    cd FlashVoteBackend
    dotnet restore
    dotnet build
    dotnet run
    ```

    The backend binds to `http://0.0.0.0:5000`. Once running, the API is available at `http://localhost:5000`.

    <Note>
      The Swagger UI is available at `http://localhost:5000/swagger` when the backend is running in development mode. Use it to explore and test all REST endpoints without writing any code.
    </Note>

    <Tip>
      Use `dotnet watch run` instead of `dotnet run` for hot-reload during backend development. The server restarts automatically when you save source files.

      ```bash
      dotnet watch run
      ```
    </Tip>
  </Step>

  <Step title="Configure the frontend environment">
    Create a `.env` file inside `FlashVoteFrontend/` to point the frontend at your local backend:

    ```bash
    # FlashVoteFrontend/.env
    VITE_API_BASE_URL=http://localhost:5000
    ```

    If this file is absent, the HTTP client falls back to its built-in default URL.
  </Step>

  <Step title="Start the frontend">
    From the `FlashVoteFrontend/` directory, install dependencies and start the Vite dev server:

    ```bash
    cd FlashVoteFrontend
    npm install
    npm run dev
    ```

    Vite prints the local URL when it starts — typically `http://localhost:5173`. Open that URL in your browser.
  </Step>

  <Step title="Test real-time updates">
    Verify that SignalR vote broadcasting works:

    1. Open the app in **two separate browser tabs** at `http://localhost:5173`.
    2. Select the same poll in both tabs.
    3. Vote on an option in tab A.
    4. Watch the charts and vote counts in tab B refresh automatically — no page reload required.

    The vote flows from the frontend through the REST endpoint (`POST /api/poll/{pollId}/vote/{optionId}`), the backend persists it and broadcasts a `ReceiveVote` event over SignalR to the poll group, and the frontend invalidates its React Query cache to re-fetch the latest results.
  </Step>
</Steps>
